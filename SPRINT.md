# 🚀 Facebook Marketplace AI Lister - Sprint 计划

**项目目标**：用户上传商品图片 → AI 自动生成列表信息 → 自动填充 Facebook 表单

**使用技术**：Vanilla JavaScript、Chrome Manifest V3、Google Gemini Vision API

---

## 📋 User Stories & Tasks

### Sprint 1️⃣: 基础架构和 UI（Phase 1-2）

#### US-1.1: Chrome 扩展基础设置 ✅
- **描述**：作为开发者，我想要一个正确配置的 Manifest V3 文件，以便扩展能正确加载到 Chrome
- **验收标准**：
  - [x] manifest.json 配置正确
  - [x] 扩展能在 chrome://extensions 中加载
  - [x] 无任何加载错误
  - [x] 图标正常显示（待添加）
- **优先级**：🔴 高
- **完成日期**：已完成

#### US-1.2: 扩展弹窗 UI ✅
- **描述**：作为用户，我想要一个简洁的弹窗，可以选择和预览商品图片，并显示当前设置的地址
- **验收标准**：
  - [x] 弹窗显示在扩展图标下方
  - [x] 有"选择图片"按钮
  - [x] 支持 jpg、png、webp 格式
  - [x] 图片预览正确显示
  - [x] 显示当前设置的地址（比如"北京" ）
  - [x] 有"分析"按钮
  - [x] 可以快速修改地址（链接到设置）
  - [x] UI 美观（清晰、易用）
- **优先级**：🔴 高
- **完成日期**：已完成

#### US-1.3: 用户地址设置 ✅
- **描述**：作为用户，我想要设置我所在的地区/城市，以便 AI 能根据当地物价生成准确的价格建议
- **验收标准**：
  - [x] 在弹窗中有地址输入框（或下拉选择）
  - [x] 支持输入城市名称（如"北京"、"上海"、"纽约"等）
  - [x] 地址保存到 chrome.storage.local
  - [x] 第一次打开时提示用户输入地址
  - [x] 可以随时修改地址
  - [x] 地址信息清晰显示
- **优先级**：🔴 高
- **完成日期**：已完成

#### US-1.4: 加载和错误反馈 ✅
- **描述**：作为用户，我想要看到加载状态和错误提示，以便了解处理进度
- **验收标准**：
  - [x] 分析时显示加载动画
  - [x] 成功显示"✅ 分析完成"提示
  - [x] 错误显示清晰的错误信息
  - [x] 可以重试操作
- **优先级**：🟡 中
- **完成日期**：已完成

---

### Sprint 2️⃣: Vision AI 集成（Phase 3）

#### US-2.1: Gemini API 集成 ✅
- **描述**：作为开发者，我想要正确配置 Gemini API 调用，以便能分析商品图片
- **验收标准**：
  - [x] API 密钥已配置
  - [x] Base64 图片正确发送到 Gemini API
  - [x] 收到 Gemini 的 JSON 响应
  - [x] 错误处理正确（超时、API 错误等）
  - [ ] 支持重试机制（最多 3 次）- 可选优化
- **优先级**：🔴 高
- **完成日期**：已完成

#### US-2.2: 通用商品分析提示词（考虑地域因素）✅
- **描述**：作为用户，我想要 AI 能分析各种商品（电子产品、服装、家具等），生成相关的标题、价格、描述，价格要根据我所在的地区调整
- **验收标准**：
  - [x] AI 能正确识别商品类型
  - [x] 生成的标题吸引人且准确（15-85 字）
  - [x] **价格范围根据用户所在地区调整**（如北京、纽约价格不同）
  - [x] 价格范围合理（基于商品新旧程度）
  - [x] 描述包含 3-5 个关键特点
  - [x] 推荐的类别正确
  - [x] 支持中文和英文
  - [x] 在提示词中清晰传达用户地址给 AI
- **优先级**：🔴 高
- **完成日期**：已完成

#### US-2.3: AI 响应解析 ✅
- **描述**：作为开发者，我想要正确解析 Gemini 的 JSON 响应，转换为可用数据
- **验收标准**：
  - [x] 解析 title、priceRange、description、category
  - [x] 数据验证（字段非空、类型正确）
  - [x] 处理意外的 JSON 格式
  - [x] 能处理特殊字符和 emoji
- **优先级**：🔴 高
- **完成日期**：已完成

#### US-3.1: 内容脚本注入和 DOM 监听 ✅

  - [x] 正确清理观察器（防止内存泄漏）
- **优先级**：🔴 高

#### US-3.2: Facebook 表单字段定位 ✅
- **描述**：作为开发者，我想要准确定位 Facebook 表单中的输入字段，以便填充数据

#### US-3.3: 自动填充功能 ✅

## 📝 注意事项

1. **API 密钥**：用户需要提供有效的 Google Gemini API 密钥
2. **Facebook 变化**：Facebook 可能会改变 DOM 结构，需要定期更新选择器
3. **API 配额**：Gemini 免费额度每天 15 个请求，超过需要付费
4. **Chrome 版本**：需要 Chrome 110+ 支持 Manifest V3
